# Coding Task Checklist

## Before Editing

- [ ] Latest user instruction is understood.
- [ ] Read-only vs implementation mode is clear.
- [ ] Applicable `AGENTS.md` or project rules are checked.
- [ ] Dirty worktree baseline is recorded.
- [ ] Existing implementation patterns are searched.

## Implementation

- [ ] Change is minimal and scoped.
- [ ] User changes are preserved.
- [ ] Public contracts remain compatible or were explicitly approved.
- [ ] Error handling and edge cases are considered.

## Verification

- [ ] Diff was reviewed.
- [ ] Relevant tests, type checks, lint, build, or smoke checks ran.
- [ ] Unverified paths and residual risks are documented.
