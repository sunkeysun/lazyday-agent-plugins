# Source Types

## 一句话或聊天内容

- 把用户原文作为 raw source，写入 `original/input.md`。
- 保留提交时间、上下文、用户原话，不要直接改写成总结。
- 加工层再抽取情绪、事件、人物、项目、任务、反思和标签。

## 普通文件

- 复制原文件到 `raw/.../original/`，记录 sha256、size、mime、原路径。
- 不要直接在原路径上修改。
- 文本类文件可抽取到 `processed/extracts/`。
- 如果文件是 iCloud 占位、未下载或 hash 无法计算，manifest 标记 `availability: cloud-placeholder`、`raw_downloaded: false`、`status: partial | blocked`，不要标记完成。

## 多文件输入

- 先判断文件之间是同一事件证据、一个逻辑文档、一个容器、还是互不相关。
- 同一事件的截图、日志、录屏、用户说明应使用同一 `batch_id`，但仍为多个 child source。
- 压缩包本身是容器 source；内部文件清单进入 extract/source map，除非用户明确要求把内部关键文件独立入库。
- 关系不明确时不要强行关联；先创建 mixed/batch source 或询问。

## 图片

- 原图完整保留。
- 记录 EXIF、尺寸、文件名、上下文。
- 如果可用，生成 OCR、视觉描述、可见实体、场景、时间线线索和 asset 索引。
- 召回图片时优先返回 raw path、缩略图路径、source_id、可见内容摘要和相关 wiki 链接。

## 音频/视频

- 原文件完整保留。
- 优先抽取元数据和转写；没有转写能力时标记 `status: partial` 并记录需要的工具。
- 对视频保留关键时间戳、可见画面摘要、转写段落和关键帧路径。
- 引用必须使用时间戳定位，例如 `#t00:03:12-t00:04:08`。

## 压缩包

- 压缩包本身作为 raw source 全量保留。
- 创建文件清单，记录内部路径、大小、hash、可疑文件、解压状态。
- 未经确认不要删除压缩包；如需解压，解压结果属于 processed 或 raw 子目录的派生文件，并关联原压缩包 source_id。

## 电子书/PDF

- 原文件完整保留。
- 抽取目录、页码、章节、引文、图片说明和高亮。
- 引用必须保留页码或章节定位。

## 日志和故障资料

- `source_type` 使用 `log`。
- 原日志完整保留，不要只留错误摘要。
- 加工层提取时间线、错误边界、环境版本、请求 ID、用户操作、关键堆栈和候选根因。
- 对故障问答，优先召回原始日志片段，再给分析。
- 如果日志与截图、录屏、压缩包同批提供，manifest 记录 `batch_id` 和 `related_sources`，方便后续组合召回。

## 网页和外部链接

- 若用户只给 URL，先记录 URL、访问时间和标题。
- 只有在用户允许抓取时才下载网页内容。
- 下载后保留原始 HTML/截图/markdown，避免只保留摘要。

## 敏感资料

- 默认 `privacy: sensitive | unknown` 时不调用外部服务。
- 输出中只展示必要片段；避免整段打印 token、凭据、身份证件、私人聊天等内容。
